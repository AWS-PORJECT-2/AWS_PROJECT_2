# 두띵(doothing) 전면 재구축 스펙 v3

> 텀블벅·와디즈·Kickstarter·Indiegogo 리서치 4건 + 현재 사이트 감사 2건 종합. 국민대 시작 → 일반 확장 굿즈 크라우드펀딩 플랫폼.
> 기술 제약: Vanilla JS(프레임워크 금지) + Express/TS + PostgreSQL. 프론트는 server가 서빙하는 정적 파일.
> 톤: 텀블벅식 화이트 배경 + 카드 그림자. 메인색 연보라(#8B5CF6)는 CTA·달성률바·배지에만 절제 사용. 이모지 금지(아이콘은 이미지/SVG). "AI로 대충 만든 티" 금지.

## 0. 핵심 의사결정 요약 (Decision Table)

| 영역 | 채택 모델 | 두띵 확정안 |
|------|-----------|-------------|
| 헤더 | 텀블벅(미니멀 3+α) 2단 | 상단줄: 로고/검색/개설/프로필 · 하단줄: 카테고리▾/홈/인기/신규/마감임박 |
| 모바일 | 와디즈 5탭 GNB | 하단 고정 탭바 5개(홈/둘러보기/개설/찜/마이) + 상단 56px 헤더 |
| 홈 | 텀블벅 3섹션+배너 | 히어로→인기→마감임박(D-5)→신규(5일)→카테고리별→푸터 |
| 상세 | 텀블벅 좌우2단 | 좌 갤러리 / 우 sticky 후원패널 / 탭6 / 리워드 재고배지 / 창작자 카드 |
| 개설 | 와디즈8 → 6단계 압축 | 요금제·유형→기본정보→목표·일정→리워드→계획·신뢰→검토. 진행률%, 승인 후 목표금액 잠금 |
| 후원 | Kickstarter 체크아웃 | 리워드선택→배송지(필수)→요약→무통장입금 안내→동의→완료(입금대기) |
| 마이/설정 | 텀블벅 분리형 | 마이(조회): 찜/내펀드/참여/최근본/팔로우 · 설정(편집): 프로필/계정/결제/배송지/알림+탈퇴 |
| 결제 | 무통장입금 1차 | 사업자명 가상계좌 → 관리자 입금확인 → 확정. 카드(토스)는 Phase 후속 |
| 환불 | 전자상거래법 준수 | "환불 불가" 약관 금지. 청약철회·하자환불 기본 제공 |

**차별 포인트:** 텀블벅은 "종료 다음날 자동결제(All-or-Nothing)"인 반면 두띵은 **무통장입금 → 관리자 입금확인** 구조다. 따라서 후원~입금확인~정산을 별도 상태머신으로 설계하고, 관리자 콘솔(승인·입금확인·삭제/환불)과 연동해야 한다.

## 1. 정보구조(IA) · 전역 네비게이션

### 1.1 사이트맵

```
/ (landing)              비로그인 랜딩
/main                    홈(로그인 후 메인) ← 큐레이션 섹션
/feed                    피드(목록): 카테고리/검색/정렬/페이지네이션
/categories              카테고리 아이콘 그리드(13+기타) — 전용 페이지 + 헤더 메가메뉴 공유
/detail?id=              프로젝트 상세
/fund-create             펀딩 개설 위저드(6단계)
/payment?orderDraft=     후원/결제
/profile                 마이페이지(탭: 좋아요/내펀드/참여/최근본/팔로우)
/settings                설정(탭: 프로필/계정/결제수단/배송지/알림 + 탈퇴)
/addresses               배송지 관리(설정 탭 인라인 + 독립 페이지 겸용)
/login, /onboarding, /terms, /privacy   인증·동의·온보딩
/support                 고객문의(폼)
/admin                   관리자 콘솔(탭: 승인큐/입금확인/삭제·환불/사용자)
```

### 1.2 데스크톱 헤더 — 2단 (≥1024px)

```
┌─────────────────────────────────────────────────────────────────────┐
│ [doothing 로고]              [🔍 검색바.................]  [펀딩 개설] [프로필▾] │  ← 1단 (h:60px)
├─────────────────────────────────────────────────────────────────────┤
│ [≡ 카테고리▾]   홈   인기   신규   마감임박                                │  ← 2단 (h:48px, sticky)
└─────────────────────────────────────────────────────────────────────┘
```

- **로고**: 클릭 시 `/main`(로그인) 또는 `/`(비로그인).
- **검색바**: 1단 중앙. placeholder "프로젝트, 카테고리 검색". Enter → `/feed?q=`. 포커스 시 최근검색어/인기검색어 드롭다운.
- **펀딩 개설**: 보라 outline 버튼. 비로그인 클릭 시 로그인 유도.
- **프로필▾**: 로그인 시 아바타 + 드롭다운(마이페이지 / 내 펀드 / 설정 / [관리자면 관리자 콘솔] / 로그아웃). 비로그인 시 `[로그인]` 링크.
- **카테고리▾**: 호버 시 **메가메뉴** 펼침 — 13+기타 아이콘 그리드(의류류 → 소품류 → 기타 그룹 정렬, 가나다 강제정렬 아님).
- **2단 네비**: 홈/인기/신규/마감임박은 `/feed?sort=` 또는 `/main#anchor`로 라우팅. 현재 위치 underline(보라 2px).
- 알림 벨(프로필 좌측): 미읽음 시 빨강 점 배지.

### 1.3 카테고리 메가메뉴 (호버 펼침)

```
┌─ 카테고리 ──────────────────────────────────────────────┐
│ 의류         [과잠] [후드티·맨투맨] [반팔티]                  │
│ 소품         [에코백] [키링·스트랩] [폰케이스] [스티커·문구]   │
│              [뱃지·배지] [텀블러·머그] [담요·패브릭]           │
│              [인형·마스코트] [액세서리]                       │
│ 기타         [기타]                                          │
└──────────────────────────────────────────────────────────┘
```
각 항목 = 아이콘(40px) + 라벨. 클릭 → `/feed?category=<slug>`.

### 1.4 모바일 (≤767px)

**상단 헤더 (h:56px, sticky top, z-index:100):**
- 일반 페이지: `[← 뒤로] · doothing(중앙) · [🔍] [☰]`
- 메인 페이지: `[☰] · doothing(중앙) · [🔔] [아바타]`
- `☰` 클릭 → **풀스크린 드로어**(카테고리 13+기타 아코디언/그리드 + 로그인/설정/문의). 데스크톱 호버 메가메뉴는 터치에서 동작 안 하므로 드로어로 전환.

**하단 탭바 (신규 도입, h:56px + safe-area, z-index:90, ≤767px만 노출):**

```
┌────────┬──────────┬──────────┬────────┬──────────┐
│  홈    │ 둘러보기  │  개설하기 │  찜    │  마이     │
└────────┴──────────┴──────────┴────────┴──────────┘
```
- 5탭(3~5 권고, 초과 금지). 아이콘 24px + 라벨 10~11px.
- 활성=`#8B5CF6`, 비활성=`#9CA3AF`. 탭 터치영역 최소 48×48px.
- `body { padding-bottom: calc(56px + env(safe-area-inset-bottom)); }` 로 본문 가림 방지.
- **충돌 규칙**: 상세 페이지의 하단 고정 후원바와 겹치므로 → 상세에서는 **탭바 숨기고 후원바만** 노출. (또는 후원바 `bottom: 56px`)

## 2. 디자인 시스템

모든 토큰은 `:root` CSS 변수로 정의하고 전 페이지가 참조한다. 페이지별 하드코딩 hex 금지.

### 2.1 색상 토큰 (연보라 팔레트)

```css
:root {
  /* Primary — 연보라 */
  --c-primary-50:  #F5F3FF;   /* 배경 틴트, hover 면 */
  --c-primary-100: #EDE9FE;   /* 배지 배경, 진행바 트랙 */
  --c-primary-200: #DDD6FE;
  --c-primary-300: #C4B5FD;
  --c-primary-400: #A78BFA;
  --c-primary-500: #8B5CF6;   /* ★ 메인 — CTA/진행바/활성 */
  --c-primary-600: #7C3AED;   /* hover/pressed */
  --c-primary-700: #6D28D9;   /* 강조 텍스트(흰 배경 위 AA 통과) */

  /* 중립(텍스트는 보라 아님 — 대비 4.5:1 확보) */
  --c-text:        #1A1A1A;   /* 본문 */
  --c-text-sub:    #4B5563;   /* 보조 */
  --c-text-muted:  #6B7280;   /* 메타(≥12px에서만) */
  --c-text-faint:  #9CA3AF;   /* placeholder/비활성 — 본문 사용 금지 */
  --c-border:      #E5E7EB;
  --c-divider:     #F3F4F6;
  --c-surface:     #FFFFFF;
  --c-bg:          #FAFAFB;   /* 페이지 배경 */

  /* 상태 */
  --c-success: #16A34A;  /* 성공/달성 */
  --c-warning: #F59E0B;  /* 마감임박 D-3 강조 */
  --c-danger:  #EF4444;  /* 마감/품절/반려/삭제 */
  --c-info:    #3B82F6;
}
```

**대비 규칙(WCAG, 감사 PART1-D):** 흰 배경 본문 텍스트에 `#8B5CF6` 직접 사용 금지(대비 미달). 본문은 `--c-text`/`--c-text-sub`, 보라는 **버튼 면·진행바·배지·아이콘**에만. 보라 강조 텍스트가 필요하면 `--c-primary-700`(#6D28D9).

### 2.2 타이포 스케일

폰트: `Pretendard, -apple-system, "Apple SD Gothic Neo", system-ui, sans-serif`. 한글 줄바꿈 `word-break: keep-all`.

| 토큰 | size / line-height / weight | 용도 |
|------|------|------|
| `--fs-display` | 32 / 1.25 / 800 | 상세 프로젝트명, 히어로 |
| `--fs-h1` | 26 / 1.3 / 800 | 페이지 타이틀 |
| `--fs-h2` | 20 / 1.35 / 700 | 섹션 제목 |
| `--fs-h3` | 18 / 1.4 / 700 | 카드/패널 제목 |
| `--fs-body` | 16 / 1.6 / 400 | 본문(모바일 input 자동줌 방지 하한) |
| `--fs-body-sm` | 14 / 1.6 / 400 | 카드 보조 텍스트 |
| `--fs-caption` | 12 / 1.5 / 500 | 메타·라벨(하한, 그 이하 금지) |
| `--fs-tab` | 11 / 1.2 / 600 | 하단 탭바 라벨(예외) |
| 달성률 % | 22 / 1 / 800 | 카드·상세 강조 수치 |

### 2.3 간격 (4px 베이스 스케일)

```
--sp-1:4  --sp-2:8  --sp-3:12  --sp-4:16  --sp-5:20  --sp-6:24  --sp-8:32  --sp-10:40  --sp-12:48  --sp-16:64
```
- 콘텐츠 래퍼 `max-width: 1200px; margin:0 auto;` 좌우 패딩 데스크톱 24 / 모바일 16.
- 섹션 상하 여백 데스크톱 48 / 모바일 32.

### 2.4 라운드 / 그림자

```css
--r-sm: 8px;   /* 인풋, 배지 */
--r-md: 12px;  /* 버튼, 카드 */
--r-lg: 16px;  /* 패널, 모달 */
--r-full: 999px; /* 칩, 아바타 */

--sh-1: 0 1px 2px rgba(16,24,40,.06);                 /* 기본 카드 */
--sh-2: 0 4px 12px rgba(16,24,40,.08);                /* hover 카드 */
--sh-3: 0 8px 24px rgba(16,24,40,.12);                /* 모달/드롭다운 */
--sh-primary: 0 6px 18px rgba(139,92,246,.28);        /* CTA hover */
```
브레이크포인트 4단 통일: **`479 / 767 / 1023 / 1279`** (리서치4). 페이지별 900/800/1024 혼재 제거 → 768이 모바일 전환 메인 분기.

### 2.5 컴포넌트 스펙

**버튼**
| 변형 | 면/테두리/텍스트 | 크기 |
|------|------------------|------|
| Primary | bg `--c-primary-500`, hover `--c-primary-600`, text #fff | h:48(데), 52(모/CTA), pad 0 20, radius `--r-md`, weight 700 |
| Secondary(outline) | bg #fff, border 1.5px `--c-primary-500`, text `--c-primary-700` | 동일 |
| Ghost | bg transparent, hover `--c-primary-50`, text `--c-text-sub` | 동일 |
| Danger | bg `--c-danger`, text #fff | 동일 |
| Disabled | bg `--c-divider`, text `--c-text-faint`, cursor not-allowed | — |
- 시맨틱 `<button>`/`<a>`만 사용(div/span 가짜 버튼 금지). 터치영역 ≥44px. 포커스 링 `outline: 2px solid var(--c-primary-400)`.

**배지(Badge)**
| 배지 | 스타일 |
|------|--------|
| 카테고리 | bg `--c-primary-50`, text `--c-primary-700`, radius full, 12px |
| 진행중 | bg `--c-primary-100`, text `--c-primary-700` |
| 마감임박(D-3↓) | bg #FEF3C7, text #B45309 |
| 성공/달성 | bg #DCFCE7, text #15803D |
| 마감/품절/반려 | bg #FEE2E2, text #B91C1C |
| 두띵 검수완료 | bg `--c-primary-500`, text #fff, 좌측 체크 아이콘 |
| 대리 운영 | bg #F3F4F6, text #4B5563 |

**카드(공통 컨테이너)**: bg #fff, border 1px `--c-border`, radius `--r-md`, shadow `--sh-1`, hover `--sh-2` + `translateY(-2px)` transition 160ms. 썸네일 `aspect-ratio` 고정으로 레이아웃 점프 방지.

**인풋**
- `<label>` 필수 연결(접근성). bg #fff, border 1px `--c-border`, focus border `--c-primary-500` + ring, radius `--r-sm`, h:48, **font-size 16px(iOS 자동줌 방지)**, pad 0 14.
- 에러: border `--c-danger` + 하단 구체 메시지("전화번호는 숫자 11자리"). 모호한 "잘못된 입력" 금지.
- 진행바: 트랙 `--c-primary-100` h:8 radius full, 채움 `--c-primary-500`. 100%↑ 시 채움 `--c-success` + "초과 달성" 라벨.

## 3. 페이지별 레이아웃 (데스크톱 + 모바일)

각 페이지 공통: 헤더(2단/모바일56) → 콘텐츠(max 1200) → 푸터. 빈 데이터 시 빈상태 일러스트 + 안내.

### 3.1 홈 `/main`

**데스크톱:**
```
[히어로 배너/캐러셀]  관리자 큐레이션 주목 펀드 1~3 (좌 카피+CTA / 우 이미지, 16:9)
[인기 펀드]           후원자수·달성률 정렬, 카드 4열 + "더보기→/feed?sort=popular"
[마감 임박]           D-5 이하, 카드 4열, 마감임박 배지 강조
[신규 펀드]           공개 5일 미만, 카드 4열
[카테고리별 인기]      13 카테고리 아이콘 그리드(7열×2줄) → 클릭 시 /feed?category=
[최근 본 프로젝트]     localStorage, 단 존재검증된 것만(버그1.2)
[푸터]
```
- **빈 섹션 숨김 규칙**: 데이터 N개 미만이면 섹션 자체 미렌더(운영 초기 흉한 빈칸 방지).

**모바일:** 히어로(풀폭, 16:9 또는 3:4) → 각 섹션 가로 스크롤 캐러셀(`scroll-snap-x`) 또는 2열 그리드 → 카테고리 4열 그리드 → 하단 탭바.

### 3.2 카테고리 `/categories` + 메가메뉴

- 아이콘 그리드: **데스크톱 7열 / 태블릿 5~6열 / 모바일 4열 / 소형 4열(gap만 축소)**. 13+기타=14개 → 7열 2줄 정확히 떨어짐(현 `repeat(7)` 버그는 13개에서 깨지지만 14개 그리드 기준이면 OK; 카드 개수 14 보장).
- 각 아이콘: 1:1 정사각 카드, 아이콘 70%, 라벨 13px(모바일 12px). 호버 시 보라 그림자 + `translateY(-3px)`.

### 3.3 피드 `/feed`

**데스크톱:**
```
[페이지 타이틀 / 검색어 표시]
[카테고리 칩 가로 스크롤]  전체 + 13개. 활성 칩 보라 fill
[정렬 드롭다운(인기/신규/마감임박/달성률) ········· 결과 N건]
[프로젝트 카드 그리드 4~5열]
```

상단 **진행률 바(%)** + 단계 인디케이터 고정. 각 단계 **임시저장**(이탈 방지), 필수 미입력 시 다음 차단.

|------|------|------|

### 3.6 후원/결제 `/payment` (3~4단계, 짧게)

상단 진행률. 안티패턴 차단(감사 PART1-A): 총액 선노출, 강제 긴 폼 금지.
```
1. 리워드(선물) 선택   ← detail에서 선택한 rewardTierId 전달(감사 4-1 결함)
2. 수량/옵션(사이즈·색상)
3. 배송지(필수)        ← 저장 배송지 선택 or 신규. 없으면 등록 유도(감사 4-2)
4. 후원 요약           ← 리워드가+배송비+총액 항상 합산(서버 재계산, 클라값 불신)
5. 결제수단: 무통장입금  ← 사업자명 가상계좌/금액/입금기한/입금자명 안내
6. 동의 체크           ← "미달성/지연 가능성·환불정책 동의"(사전체크 금지)
7. 완료                ← "입금 대기 → 관리자 확인 후 확정" 상태 명시
```
**모바일:** 단계별 풀스크린 + 하단 고정 "다음/후원 완료". 숫자 입력은 숫자 키보드.

### 3.7 마이페이지 `/profile` (조회 중심)

### 3.9 관리자 콘솔 `/admin`

## 4. 핵심 카드/티어/창작자 카드 구성요소

### 4.1 프로젝트 카드 (피드/그리드)

위→아래:
1. 대표 이미지 (aspect 4:3 또는 16:9 고정. 호버 시 2번째 이미지 전환 — 선택)
2. 좌상단 카테고리 배지 + (선택) "두띵's Pick" 큐레이션 배지
3. 프로젝트명 (`-webkit-line-clamp:2`, `escapeHtml()` 렌더, XSS 방지)
4. 창작자/주최 (학과·동아리·닉네임 + 작은 아바타)
5. **달성률 % (가장 크고 진하게)** + 가는 진행바
6. 모인 금액(또는 후원자 수) · D-N
7. 상태 배지: 진행중 / 마감임박(D-3↓ 경고색) / 성공 / 마감

### 4.2 리워드(선물) 티어 카드

- **가격**(크게) + 얼리버드 시 정가 취소선
- **티어명**("키링 단품", "풀세트+스티커 증정")
- **구성 리스트**(불릿: 본품×1, 스티커×2…) — 사용자 입력 그대로
- **옵션 셀렉트**(사이즈/색상 — 필요 시)
- **남은 수량**: "12개 남음" → 임박 강조 → **품절 시 카드 전체 회색 비활성 + "품절" 라벨, 클릭 불가**
- **예상 발송일**(배송 신뢰 핵심)
- 선택 후원자 수("34명 후원" — 소셜 증거)
- 권장 3~5개(가격 낮은→높은). 가격·재고 **반드시 서버 검증**(CLAUDE.md 규칙7).
- 승인 전 자유 수정 / 승인 후 재고·노출만 조정(Kickstarter/Indiegogo 절충).

### 4.3 창작자 카드

- 프로필 사진 + 이름/학과
- 누적 후원액 · 개설 펀드 수 · 이전 프로젝트 이력(미배송 이력 투명 공개 — 신뢰 신호)
- **[팔로우]** 버튼 + **[창작자에게 문의]** 버튼
- 학교 이메일 인증 배지(@kookmin.ac.kr) — 신뢰 신호
- 대리 운영 펀드는 "대리 운영" 배지

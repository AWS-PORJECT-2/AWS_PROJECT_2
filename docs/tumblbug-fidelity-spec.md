# 두띵 — 텀블벅 충실도 빌드 스펙 (참조 스크린샷 7장 기반)

> 목적: 두띵을 **텀블벅 수준의 실서비스 외형**으로 전 페이지 재구축.
> 이 문서가 단일 진실. 작업자는 스크린샷을 볼 수 없으니 **여기 적힌 레이아웃·치수·컴포넌트를 그대로** 구현한다.
> 브랜드색은 **연보라(#8B5CF6)** 유지하되 **헤더는 흰색**, **대표 전환 CTA(후원하기/제출/등록완료)는 검정(#1A1A1A)**,
> 달성률·강조 숫자는 **보라(#7C3AED) 굵게**, 보조 강조는 보라 배지. (기존의 보라 그라데이션·통보라 헤더 = 폐기. 그게 "AI 딸깍" 느낌의 주범.)

## 0. 공통 원칙
- 토큰: `tokens.css` 변수만 사용. 페이지별 하드코딩 hex 금지.
- 콘텐츠 폭 `--wrap-max:1200px`, 좌우 패딩 24px(모바일 16px).
- 폰트 Pretendard. 본문 16/14, 캡션 12. 줄간 1.5~1.6.
- 카드/입력 라운드 12px, 큰 영역 16px. 그림자 절제(--sh-1 기본, hover --sh-2).
- 이미지 에셋 `/assets/`(전부 존재): logo.png, logo-mark.png, 카테고리 13종(jacket·tshirt·ecobag·hoodie·keyring·phonecase·sticker·badge·tumbler·fabric·doll·accessory·etc).png, empty-feed/likes/funds/backings.png, error-404.png, hero-main.png, hero-mobile.png, onboard-illust.png, trust-badge.png.
- **반응형 분기 통일**: 1023↓(태블릿), 767↓(모바일), 479↓(소형). 모바일은 하단 5탭바(.dt-tabbar).

## 1. 공통 헤더 (.dt-hd) — 흰색 2단

## 3. 페이지별

### 3.1 홈 main.html (스크린샷1)
헤더 → (콘텐츠 1200px) → 2단 그리드:
- 좌(약 62%): **히어로 배너** — `/assets/hero-main.png`를 큰 배너로(라운드 12). 위에 "마감 임박" 등 배지+카피 오버레이는 옵션(데이터 있으면 인기 1위 프로젝트로 캐러셀, 없으면 hero-main 정적 + "두띵에서 시작하세요" 카피+CTA). 하단우측 캐러셀 화살표(데이터 2개+일 때).
- 우(약 38%): **RankList(2.2)**.
- 아래(좌폭 풀): **카테고리 아이콘 행** — `/assets/<key>.png` 실제 아이콘 8개+ 가로 스크롤/그리드, 라벨 하단. (CategoryGrid 재사용, 실이미지 우선)

모바일: 1단(히어로→랭킹→카테고리→그리드), 하단 탭바.

### 3.2 프로젝트 올리기 fund-create.html (스크린샷2) — ★구조 전면 교체
**진입 즉시 AI 가상피팅 금지.** 흐름:
1. **(첫 화면) "무엇을 만들까요?" 유형/카테고리 선택** — 카드 그리드(과잠/후드티/키링/폰케이스/텀블러/스티커/…/기타, 각 아이콘+라벨). + 진행방식 선택(직접 개설 / 대리 개설). 선택 후 "다음".
2. **프로젝트 작업실(워크스페이스)** — 상단: 대표이미지 썸네일(없으면 회색 플레이스홀더) + [카테고리] + 제목(미입력시 "내 프로젝트") + "기획중·N% 완료" pill. 아래 **섹션 카드 그리드**(각 카드 = 아이콘+제목+"N% 작성완료", 클릭 시 해당 폼 패널 오픈):
   - 기본 정보(제목/한줄소개/대표이미지)
   - 목표 금액 및 일정(목표수량or금액/마감일/예상발송)
   - 선물 구성(리워드 티어 — 기존 reward composer 재사용)
   - 프로젝트 계획(스토리 본문 — 기존 content composer 재사용)
   - 창작자 정보(소개/연락)
   - 신뢰와 안전(주의사항 체크)
   - **별도 버튼: "🎨 AI 가상피팅 생성"** → 별도 모달/페이지(design-tryon.html 또는 모달)에서만 동작. 결과 이미지를 대표이미지로 사용 가능. **워크스페이스 메인 흐름에는 임베드하지 않음.**
3. 모든 섹션 충족 시 하단 "검토 후 개설하기" → 검토 → 제출(POST /api/groupbuys).
- 진행률(%)은 채워진 섹션 비율로 계산해 상단 pill·각 카드에 표기.
- 기존 제출 페이로드(title/category/description/contentBlocks/targetQuantity/deadline/rewardTiers/mode/delegate*)는 유지. UI만 워크스페이스로 재배치.

### 3.3 상세 게시글 detail.html (스크린샷3·4)
**상단(2단):**
- 좌(50%): 대표 이미지(정사각, 라운드12) + 하단 썸네일 스트립(있으면).

**하단(2단):**
- 좌: 탭(프로젝트 계획·업데이트·커뮤니티·후기) + 서브탭 pill(소개·선물 설명·예산·일정·팀 소개·신뢰와 안전) + 스토리 본문(contentBlocks 렌더, 이미지 풀폭).
- 우(sticky): 창작자 카드(아바타·좋은창작자·이름·마지막로그인·팔로워·만족도/누적후원자/누적후원액·창작자 문의/팔로우) + **선물 선택**(리워드 리스트: 금액+ / 제목 / 설명 / 선택 버튼) + TOP 플로팅 버튼.
- 모바일: 1단, 하단 고정 "후원하기" 바.
- 데이터: 기존 /api/groupbuys/:id 응답(제목/카테고리/이미지/설명/contentBlocks/rewardTiers/현재·목표/마감) 사용. 통계(만족도 등) 없으면 항목 숨김.

> 사용자가 와디즈가 두띵에 더 가깝다고 확정. 위 텀블벅 스펙과 구조는 거의 동일(둘 다 리워드형 크라우드펀딩)하며, 아래 와디즈 디테일로 보강한다.

## A. 홈 (와디즈 홈)
- 헤더 우측에 **프로젝트 만들기** = 외곽선 버튼(이미 구현). 상단 원형 카테고리 아이콘 행(우리 카테고리 13종 원형) + 그 아래 텍스트 카테고리 메뉴(선택).
- 좌측 히어로 캐러셀 + 우측 **"실시간 베스트" 랭킹**(= 우리 RankList, 이미 구현). 랭킹 상단에 탭(전체/카테고리 칩) 선택 가능.
- "취향 맞춤 프로젝트"/"주목할 만한" 카드 그리드(달성% 강조색).

## B. 프로젝트 만들기 = 와디즈 "메이커 스튜디오 / 작성 현황" 모델 (★스펙 §3.2 이걸로 확정)
- 진입: **프로젝트 유형 선택**(펀딩/프리오더/… → 우리는 "직접 개설/대리 개설"+카테고리) + 목표금액/카테고리. (AI 가상피팅은 진입에 노출 금지, 별도 버튼/창)
- 이후 **"작성 현황" 페이지** = 세로 카드 리스트, 각 항목 = [아이콘 ○] 제목 + "작성 전/작성 완료" 상태 + 우측 **"작성하기" 버튼**. 항목: 기본 정보(서비스 요금은 우리 무관 → 생략 가능) / 프로젝트 정보(유형·목표금액·카테고리·배송유형) / 스토리(본문) / 리워드(선물 구성) / 정책(환불·정보고시·A/S) / 메이커 정보(이름·프로필·문의 이메일·SNS).
- 각 "작성하기" → 해당 폼 패널(또는 서브페이지)로. 작성 완료 시 카드 체크 + 상단 진행바.
- 하단 **"오픈 예약하기"**(모든 필수 작성 완료 시 활성) = 제출(POST /api/groupbuys). 비활성 시 회색.
- 좌측에 작은 사이드바(프로젝트 작성/일정/…)는 선택. 모바일은 카드 세로 리스트만.

## C. 상세 (와디즈 상세)
- §3.3과 동일. 탭: 스토리/업데이트(새소식)/커뮤니티/(후기). 우측 패널: 달성%·남은기간·모인금액·참여자수 → **"펀딩하기" 버튼**(우리는 검정 또는 보라). "N명이 지지서명했어요"류는 우리 데이터 없으면 생략. 리워드 선택 리스트(가격+제목+남은 수량+이미지).

## D. 마이페이지 (와디즈 마이) — §3.4 보강
- 좌측 사이드바(최근 본/팔로잉/간편결제 설정/문의내역/프로젝트 만들기/설정/…) + 메인: 인사("OOO님, 안녕하세요") + **스탯 카드 행**(펀딩+ 개수 / 스토어 / 지지서명 / 알림신청 + 포인트/쿠폰) + "최근에 봤어요" 카드. 우리 데이터로 채우되 없으면 0/빈상태.

## E. 설정 (와디즈 설정) — §3.5 대안 허용
- 와디즈식 **세로 리스트**(아바타+연필 편집 / 닉네임 / 이메일 / 국가·지역 / 비밀번호 / 알림 / 전화번호 / 생일 / SNS 계정 연동)도 가능. 탭형(프로필/계정/결제수단/배송지/알림) 또는 리스트형 중 깔끔한 쪽. 각 행 클릭/변경 동작 실연결.

## F. 전 기능 동작 (사용자 강조: "모든 기능들도 다 동작하게")
- 모든 버튼/탭/폼이 실제 동작해야 함. 더미 링크/죽은 버튼 금지. 미구현 기능은 노출하지 말거나 "준비 중" 명시.
- 제출·후원·찜·팔로우·프로필수정·배송지·로그아웃 등 기존 API 보존 + 연결 확인.
